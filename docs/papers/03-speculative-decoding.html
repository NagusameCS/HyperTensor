Composing Compression: Geodesic Speculative Decoding and Attention Residuals

By William Ken Ohara Stewart (NagusameCS) · HyperTensor Research, 2026

Drafting

Paper 3 is the most exploratory of the four. It composes ideas already implemented in the runtime (geodesic speculative decode, AttnRes) and asks whether they actually help once the compression of Papers 1\u20132 is in place. The benchmark pass that produces the headline numbers has not been run yet.

\u00a70 \u2014 Abstract (planned)

What this paper will say

Compression and inference tricks compose poorly in general. We characterise two specific compositions that the HyperTensor runtime already supports: (a) using a GP-compressed model as the draft generator in speculative decoding against the full-precision baseline (llm_generate_geodesic_speculative), and (b) layering Block Attention Residuals (AttnRes) on top of compressed attention to test whether the depth-memory mechanism survives rank reduction. We report acceptance rates, end-to-end throughput, and quality deltas, and we flag the cases where the composition is a wash or a loss.

Outline

Sections, in order

1. Why compose

Speculative decoding wins by amortising the cost of verification across multiple accepted tokens. AttnRes wins by mitigating PreNorm dilution. Compression saves bandwidth. Each is well-studied alone; the question is whether they fight each other. Concretely: a high-compression draft model produces drafts that diverge from the verifier's distribution, dropping acceptance rate. AttnRes adds a softmax over block summaries, which shifts the weight-magnitude profile that low-rank decomposition is exploiting. Both interactions are non-obvious.

2. Geodesic speculative decoding (already in runtime)

The --ott-speculative mode runs the GP-compressed model as a drafter and the full transformer as a verifier. Drafts produced by the geodesic-first decode path are rejection-sampled against the full distribution. We characterise:

  • Acceptance rate vs draft length and vs compression rank.
  • End-to-end tok/s vs baseline (full transformer) and vs Paper 1's compression-only configuration.
  • The --no-verifier ablation: drafts emitted directly. How much quality is lost in exchange for the verifier's cost.

Implementation exists; benchmark pass to be run on EC2 (compute approved).

3. Attention Residuals at low rank

The runtime exposes --attnres (default strength 0.35). We compare PreNorm vs AttnRes at three compression levels: uncompressed baseline, $k=1024$ (Paper 1's configuration), and a heavier setting. Hypotheses to test:

  • AttnRes's depth-memory benefit survives compression (good case).
  • AttnRes's softmax-over-block-summaries amplifies the rank-reduction error (bad case).
  • Hidden-state magnitude growth (the Kimi paper's $\sqrt{L}$ envelope) interacts with the compressed Frobenius error in a measurable way.

Implementation exists; PPL + perplexity-by-depth measurements pending.

Brief section on --axex-kv: applying the same projection basis to the KV cache. This is a memory-footprint result more than a throughput result and fits naturally as a sub-section here.
